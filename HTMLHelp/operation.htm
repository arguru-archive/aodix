Aodix Operation

This section guides you in the process of creating, developing and finishing your project
Please open the Quick Shortcut List and print it for quick reference

Please check the following items :

- Aodix is installed and running
- that there's at least one VST Instrument installed in a folder (where VST plug-ins should be)
- that this VST folder is properly specified in the Aodix Configuration Menu
- that the PC keyboard type is properly specified in the Aodix Configuration Menu
- that the ASIO driver is properly specified in the ASIO Configuration box
- that your ASIO subsystem is properly installed and running (read the audio card's docs,
   or use an ASIO/WDM wrapper if your sound card don't support natively ASIO)
- that the Master Output pins in the "Routing" view are assigned to audio outputs.
- that you have the Quick Shortcut List printed for quick reference

All those steps are detailed in the Getting Started section.
If all is ok, you may go further...otherwise, please read the Troubleshooting Checklist.

1. First step : loading plug-ins to instance slots and wiring

Decide about what VST instruments and effects you are going to use. First, let's worry about VST instruments.
Switch from "Sequencer" to "Routing" view (Seq. Edit section), point mouse at any empty slot of the VST Instance List, and click. If everything is properly configured, under the mouse you will see a small tree listing that reflects the structure of your VST Plug-ins folder. Select the plug-in you wish to load from there.

As soon as the plug-in is loaded, it will appear as a box object (since we are looking at the "Routing" view. If it is not automatically wired to the Master Output, draw the wires as explained in the  "Routing" view section.

In the case of a multi-output VST instrument, every output pin will be sequentially wired to a pin of the Master Output. Be careful because if some used Master Output pins are not assigned to audio outputs, you could hear no audio. So make sure they are assigned to audio outputs, or change the way the wires are connected...

We recommend that you begin with simple schemes.
Schemes including VST effects are explained in Step 4 : Mastering and Finishing Project.

Now that basic wiring is ok, you may begin to record your tracks !

2. Step 2 : recording first tracks

2.1. case 1. load a midi project
Use the importation function from "File > Import MIDI File ..." menu item.
You will be asked if you want to spread channels over several instances. Please report to section Menu Items.
Be careful : the file you import will be passed through the Midi-Mask. If you want a full import, just make sure that all the flags of the Midi-Mask are set ! (refer to the Edit section).
Also, Aodix will import the PPQN setting from the file. Do not change it.

2.2. case 2. begin recording from scratch
First decide of the PPQN value (Audio master). It is not recommended that you change it in the middle of the progress of your project.
Default is 192. And we strongly recommend that you leave it to 192 !
This is a setting for advanced users, and is intended to increase/decrease midi precision.

Set project's Time Signature (Audio Master). The most used signature is 4:4.

Set project's Tempo (Audio Master). If you will only use one tempo for the whole project, then you just need to set it once. But, if you will be changing it within the project, then you will need to set it and to record it as automation, at the beginning of the tune, and whenever it is intended to change (read Note 2 in the VST Parameters section).

Now, let's set quantize value (Edit section). Use the default 16th setting, or any other adapted to your project. You may change this setting whenever you want. Sometimes when recording you may like to set it to "free". The quantize setting only acts as a grid to which inputted events stick.

When composing your tracks, you will usually switch between these operating modes (Audio Master) :

With REC on :

Live on+ Play : traditional "online" mode where you watch timeline scrolling and
                      record at the transport cursor position.

Live off + Play : special offline mode in which you record at any position you want (static)
                       meanwhile the song is playing/looping at transport position somewhere
                       else. It means that the transport cursor and the edit cursor are no longer
                       the same cursor.

STOP (live on or off): offline mode with transport stopped. It's the classical editing mode for
                               programming (typing in notes).

Press "loop" if you wish to play and record a loop inside the pattern.

Switch the "Over" button off (Edit section) if you wish to record more than one event at the same time (like chords). Read about other buttons in the (Edit section). We recommend you leave "Step" on, in the beginning. Then you may find it useful to remove it.

3. Step 3 : Arranging Project

3.1 Record, type-in notes and arrange

Record your tracks by combining all possibilities (online/offline, play/stop, step on/off, etc).

Use your midi keyboard and your PC keyboard.
If operating with PC keyboard, you may find changing the "Pre-release" setting useful when programming a track.
This setting is the value (in midi points) by which the event will be shortened. It has no effect on events incoming from a midi keyboard.

For arranging, you need to read carefully all the Sequencer section.

You may find it useful to select a block and copy/paste it. For instance, for drum programming, you will probably worry about one measure, then replicate it to all following measures. Afterwards, you would modify some events (move/resize) of some measures. Moving and resizing objects are quantize-free operations, because they are intended for "humanized" results.

You may also use the special event "jump" (key 1). This event allows you to jump over the area defined but the "jump" object (which is resizable). This is useful if you wish to shorten a measure.

Save your project several times while composing (Menu Items). Some VST plug-ins can generate instability and crash, even though they are very good and professional.

When you achieve something consistent, you can think of modulating some parameters (volumes, filter cut-offs etc..).

3.2 Tweak parameters

In order to tweak a plug-in's parameters, you may proceed using midi events if the plug-in you are using supports midiCC (midi Continuous Controllers), or using VST automation (supported by almost all plug-ins).

Pros and cons of both methods :

MidiCC recording : what you actually record is the midi event incoming from your keyboard. So you need a midi keyboard for that. Then this midi event is interpreted by the plug-in only if the plug-in supports midiCC. If this is the case, the plug-in will not always allow you to map the knobs as you wish. But if the plug-in allows it, then it uses some sort of internal matrix that you can edit. What happens if the matrix is modified for another project ? What happens if you play the tune on another computer which has the same plug-in but not same matrix ? You may not be able to play your tune as it is intended. The only advantage of this method is that it is not subject to audio driver latencies, which in all cases is extremely low in Aodix.
=> Record midiCC : just press REC and move the knobs and faders on your midi keyboard. Of course, in Midi Mask, you must set the "B" flag.

VST automation : first, unlike midiCC, you can record VST automation with your midi keyboard and with your mouse !! Then, recorded VST automation events carry the accurate information of the plug-in's parameter that is being modified. There is a subsystem in Aodix that allows you to assign a physical knob/fader of a midi keyboard to a parameter of your plug-in. Even though you change the assignments later, the recorded automation events will have exactly the same meaning. In addition, unlike midiCC, you may assign a physical knob/fader of your midi keyboard to one or many parameters in one or many plug-ins at the same time.
=> Record VST automation : first unset the "B" flag of the Midi Mask to avoid midiCC interference. Then press AUT. Open the editor of the plug-in (interface) and move buttons with your mouse. The movements are recorded. If you wish to assign a physical knob/fader of your midi keyboard, refer to section : Interface Detailed Documentation - VST Parameters

3.3 Use Pattern Calls

Although you may write your project as one single sheet and ignore this feature, we suggest you try it once you have good knowledge of Aodix operation.

A pattern call is intended to allow you use part of a pattern as a single event in another pattern.

For instance, if you are working on pattern 00, switch to pattern 01, and arrange a loop containing one measure of drum/percussion events. If this measure does not begin at 000:0, then you will need to create a marker at the beginning of the measure.
Now return to pattern 00. Set "Mode" value to "Pattern" (Edit section). Press "REC" button.
Press any letter key on our PC keyboard. This will create a "Pat" event. You will need to browse to the first parameter and replace it with the number of the pattern that is being called (in our case, it's "01"). Then browse to the second parameter and type the number of the marker from which you want to begin the call (or "00" if the measure is located at 000:0). Then set the two last parameters to scale the called events. You may move and resize the pattern call.

Cues, jumps and Stop Marker from the called pattern are ignored in a pattern call event.
Tempo changes from the called pattern are applied to global tempo.

Read more about pattern calls in the Sequencer : Event Syntaxes section. You may create as many areas to be called in a pattern as long as you have enough markers to define their beginning point (point 000:0 and 255 markers you can place anywhere else).

Note 1: the Start/End Cues and the Stop Marker that are part of a called area are ignored in the pattern call.

Note 2 : you may not use pattern calls recursively : if the area you are calling contains itself a pattern call, the latter will be ignored.

4. Step 4 : Mastering and Finishing project

We recommend that you finish the arrangement with a Stop Marker, wisely positioned some measures after your the event, so that the audio stream can decay before stop.

It is time to add some effects to your project in the VST Instances List, like delays, reverbs, equalization, compression, and more...

Add as many effects as you need and wire them so as the finally converge to the Master Output. Do not create circular references. You may decrease the gain of a wire but clicking on the arrow and dragging the slider down.

Watch the CPU level (Audio Master), since some plug-ins have high CPU load consumption.

Before rendering, You should play the whole song for mastering purpose. Add some mastering effects at the bottom of the global routing scheme. You may use the "Stardust" plug-in supplied with Aodix.

You may also find that it is wiser to chose and load mastering plug-ins at the very beginning of a project. Keep this in mind for your next projects !

Don't forget to save your work.

5. Result : Rendering project

Now it's time to write a file that you can later convert/encode to another file format with your audio editing software.

Use the "File > Bounce..." menu item (refer to section Menu Items).

If you choose to bounce Cue Range, then Aodix will render the loop within the cues.
If not,
    if there is no Stop Marker in the pattern, Aodix will not even begin rendering your file.
    if there is a Stop Marker, then Aodix will begin rendering your file from position 000:0 and
    will stop rendering when the Stop Marker is reached. You may also stop prematurely by
    clicking on "Stop".

The output file is a multi-channel 32-bit float IEEE file or a set of mono-channel 32-bit float IEEE files (to chose, tick or untick the "Multifile" checkbox).
Most audio editing applications import stereo (2-channel) 32-bit float IEEE files. Other import any number of channels (for details, refer to section Menu Items).
.

This format has been chosen because it doesn't degrade the quality of Aodix processed data.
